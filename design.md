# 设计说明

## 1. 技术选型

### 1.1 总体原则

- 单仓库管理服务端、桌面客户端、共享模型、测试与技能集成。
- 优先选择 Python 技术栈，降低跨平台与测试成本。
- 桌面客户端采用可跨平台的 Qt for Python（PySide6），兼顾 Windows / Linux 桌面能力与桌面小组件实现。
- 服务端采用 FastAPI + Uvicorn，提供清晰的 REST API。
- 数据库采用 SQLite + SQLAlchemy + Alembic 风格迁移管理。
- 测试使用 pytest，UI/集成验证采用 pytest + Qt 测试辅助。

### 1.2 模块划分

仓库建议结构：

```text
personal-task-server/
  requirements.md
  design.md
  tasks.md
  pyproject.toml
  src/
    personal_task_server/
      server/
      client/
      shared/
      skills/
  tests/
    unit/
    integration/
    ui/
  fixtures/
  scripts/
```

- `server/`: FastAPI 服务端、认证、中间件、账单处理任务。
- `client/`: PySide6 客户端，包含主窗口、日历小组件、任务详情弹窗、账单视图、连接配置。
- `shared/`: 枚举、DTO、数据库模型、配置与通用工具。
- `skills/`: 面向 Agent 的任务和财务访问封装。

## 2. 服务端设计

### 2.1 API 分层

- `routers/`: 路由层，按任务、账单、配置、健康检查拆分。
- `services/`: 业务逻辑层，负责任务状态联动、账单标准化、合并、分类。
- `repositories/`: 数据访问层，避免路由直接操作 ORM。
- `security/`: TLS/证书校验策略、API Key/客户端证书映射、审计日志。

### 2.2 任务模型

核心实体：
- Task
- TaskSubItem
- TaskStatusHistory
- TaskTag

设计要点：
- 任务状态使用枚举。
- 子项支持独立状态与排序号。
- 状态变化统一走 service，以确保历史记录完整。
- 日期查询支持按天、周、月聚合，用于日历标记展示。

### 2.3 账单模型

核心实体：
- BillImportJob
- RawTransaction
- NormalizedTransaction
- MergedTransaction
- TransactionCategory
- AccountSource
- MerchantAlias

设计要点：
- 原始交易永久保留。
- 标准化交易与原始交易一对多或一对一映射。
- 合并结果单独存储，便于回滚和审计。
- 模型建议结果和最终人工确认结果分开保存。

### 2.4 智能分析流程

账单处理流水线：
1. 导入原始文件
2. 解析为原始交易记录
3. 标准化字段
4. 规则去重与相似候选聚合
5. LiteLLM 辅助判定重复/分类/商户归一化
6. 写入合并结果与分类结果
7. 生成月度统计

当 LiteLLM 不可用时：
- 回退到规则模式
- 记录回退原因
- 前端展示结果来源

## 3. 客户端设计

### 3.1 UI 结构

客户端包含以下主要界面：
- 启动配置页
- 主窗口
- 桌面日历小组件
- 日期任务弹窗
- 任务详情与编辑窗口
- 账单汇总窗口
- 连接与证书配置窗口

### 3.2 日历小组件

支持以下模式：
- 月视图主组件
- 周视图组件
- 紧凑悬浮视图

设计要点：
- 通过统一 ViewModel 提供日期标记数据。
- 透明度、主题、置顶等作为可持久化配置。
- 点击日期弹出任务列表，并允许快速新增任务。

### 3.3 与服务端通信

- 使用异步 HTTP 客户端访问服务端。
- 支持 HTTPS 证书校验。
- 支持加载本地客户端证书或受控 Key 配置。
- 失败场景统一反馈到状态栏和弹窗。

## 4. 安全设计

### 4.1 推荐方案

MVP 采用 HTTPS + 服务端证书校验，预留 mTLS 扩展点。
如果用户后续需要 SSH 通道模式，可在客户端配置中增加 SSH 隧道拨号方式。

### 4.2 安全原则

- 默认拒绝未认证访问。
- 配置层不提供跳过证书校验的默认能力。
- 敏感字段展示脱敏。
- 所有关键写操作写入审计日志。

## 5. Skill 设计

### 5.1 任务 Skill

提供：
- 查询某日任务
- 查询任务详情
- 创建任务
- 更新任务
- 更新任务状态
- 管理子项

### 5.2 财务 Skill

提供：
- 导入账单
- 查看月度汇总
- 查看交易明细
- 查看重复交易
- 触发重新分析

Skill 层直接请求服务端 API，并从环境变量或配置文件读取服务端地址和证书信息。

## 7. MVP 边界决策

为了确保当前环境能完整开发与验证，MVP 做以下实现决策：

- 服务端 API 使用 REST，不引入 gRPC。
- 桌面客户端先实现月视图、周视图和浮窗三种可切换样式。
- 账单数据导入先支持 CSV/TSV/通用表格文本，后续再扩展专有导出格式适配。
- LiteLLM 提供可插拔接口与 mock/fallback，确保无真实账单数据时也能完成功能验证。
- Agent skill 先以 Python 封装和 CLI/函数调用适配形式交付，后续可再转成更具体的 OpenClaw skill 包。

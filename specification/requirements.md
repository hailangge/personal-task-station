# personal-task-station 本轮需求说明

## 1. 本轮目标与范围

本轮优先级调整为 **任务管理 + 日历视图 + 部署交付**。账单/财务功能保持现有能力，不继续扩展；除解除测试或耦合阻塞外，不新增财务功能。

必须交付：
1. 完整可用的服务端任务管理能力。
2. 面向日历视图的日期/月份聚合接口。
3. 客户端任务 UI 与日历视图主流程。
4. Linux 服务端 Docker/Compose 部署能力。
5. Linux 部署/打包脚本与 Windows 客户端部署/打包脚本；Windows 优先采用可落地的 PyInstaller 免装 Python 交付方案。
6. 与上述范围一致的测试与文档。

## 2. 功能需求

### 2.1 服务端任务管理

任务字段至少包括：`id`、`title`、`description`、`scheduled_date`、`start_time`、`due_time`、`status`、`priority`、`tags`、`is_pinned`、`notes`、`created_at`、`updated_at`。

任务状态必须支持：
- `scheduled`（预约）
- `in_progress`（处理中）
- `blocked`（挂起）
- `completed`（完成）
- `cancelled`（取消）

服务端 API 必须支持：
- 创建任务
- 查询任务列表，支持日期、状态、关键字过滤
- 查询任务详情
- 编辑任务
- 删除任务
- 修改任务状态并记录状态历史
- 查询任务状态历史
- 新增、编辑、删除、重排子项
- 快速切换子项完成状态

### 2.2 日历聚合接口

服务端必须提供日历聚合接口，支持客户端按日期范围或月份获取任务标记。

聚合结果每个日期至少包含：
- 日期
- 任务总数
- 未完成/进行中/已完成/取消数量
- 是否存在置顶任务
- 最高优先级
- 用于客户端标记的状态摘要

日历接口必须能支持：
- 月视图渲染日期标记
- 点击日期后加载当日任务
- 从日期弹窗快速新增任务

### 2.3 客户端任务 UI / 日历视图

客户端必须提供清晰的主要用户路径：
- 首次启动或连接失败时配置服务端地址/API Key/证书路径，并可测试连接。
- 查看任务列表，支持今日、本周、本月、状态和关键字过滤。
- 创建、编辑、删除任务。
- 修改任务状态。
- 查看和编辑子项。
- 查看状态历史。
- 月历视图显示任务日期标记。
- 点击日期弹出当日任务列表，支持快速新增/编辑/状态切换。

UI 在无 PySide6 或无图形环境时应保持可测试的非 GUI 逻辑；自动化测试可聚焦 widget/view-model 行为。

### 2.4 部署交付

#### Docker 服务端部署

必须包含：
- `Dockerfile`
- `docker-compose.yml`
- entrypoint 或等价启动脚本
- `.dockerignore`
- 文档说明
- 最小可验证运行路径：构建镜像、启动服务、健康检查/API 验证。

容器内服务端应支持通过环境变量配置：
- 数据库路径
- API Key
- Host/Port
- TLS 证书路径（如启用 HTTPS）

#### Linux 脚本

必须提供可执行脚本：
- Linux 服务端部署脚本，能准备目录、配置环境、构建/启动 Docker Compose。
- Linux 客户端或项目打包脚本，能在 Linux 上生成可分发产物或明确完成本机安装包/压缩包。

#### Windows 客户端脚本

必须提供 Windows 客户端打包/部署脚本：
- 优先 PyInstaller/Nuitka。
- 目标为尽量免装 Python 的客户端交付。
- 若当前环境无法实际执行 Windows 打包，脚本仍应自洽，并在文档中说明需在 Windows Runner/主机执行。

## 3. 非功能与边界

- 不继续扩展账单、邮件导入或财务分析范围。
- API 错误返回应清晰，关键输入需校验。
- SQLite 为默认存储。
- 保持现有测试通过。
- 新增部署脚本应可重复执行且具备基本错误处理。
- 不在本轮直接推送远端；完成后交接给 github-assistant 提交推送。

## 4. 验收标准

- 规格文档、实现、测试一致。
- 任务管理服务端主流程可通过单元/集成测试验证。
- 日历聚合接口与客户端日历标记可通过测试验证。
- 客户端任务 UI 的主要交互路径有自动化测试或可运行 smoke 验证。
- Docker 部署文件齐全，并在当前 Linux 环境完成最小验证（至少配置检查/构建或可解释的受限验证）。
- Linux/Windows 部署脚本存在、可读、具备执行说明；Linux 脚本在当前环境完成语法或 dry-run 验证。
- Kimi 独立复核任务主流程、日历 UI/接口、Docker 部署、Linux/Windows 打包脚本并留下结论。

# 支付渠道交易信息拉取接口调研报告

## 1. 概述

本报告调研了国内六大主流支付/消费渠道的交易数据自动化获取方式：
- 微信支付（WeChat Pay）
- 支付宝（Alipay）
- 淘宝/天猫（Taobao/Tmall）
- 京东（JD）
- 拼多多（PDD）
- 招商银行（CMB）

调研范围包括：官方开放 API、第三方 SDK、邮件对账单、手动导出等可行方案。

---

## 2. 微信支付（WeChat Pay）

### 2.1 官方开放 API（商户平台）

| 项目 | 详情 |
|------|------|
| 平台名称 | 微信支付商户平台（pay.weixin.qq.com） |
| API 文档 | https://pay.weixin.qq.com/wiki/doc/apiv3/apis/chapter3_1.shtml |
| 核心接口 | `GET /v3/bill/tradebill` |
| 接口功能 | 下载交易账单（按日） |
| 数据格式 | CSV / GZIP 压缩 |
| 权限要求 | 需开通微信支付商户号 |
| 个人可用 | **否**，需要营业执照等商户资质 |
| 认证方式 | 商户 API 证书（mTLS）+ APIv3 密钥 |

### 2.2 个人用户方案

微信支付**没有面向个人消费者的开放 API**。个人用户获取账单的方式：

#### 方案 A：微信 App 账单导出（推荐）

| 项目 | 详情 |
|------|------|
| 操作路径 | 微信 → 我 → 服务 → 钱包 → 账单 → 常见问题 → 下载账单 |
| 导出类型 | 「用于个人对账」或「用做证明材料」 |
| 导出格式 | Excel（个人对账）/ PDF（证明材料） |
| 时间范围 | 最长 1 年 |
| 邮箱接收 | 填写邮箱后，微信发送下载链接到邮箱 |
| 自动化 | 需手动操作，无法定时自动导出 |

#### 方案 B：商户号 API（仅适用于商户）

如果用户有微信支付商户号，可通过 API 自动拉取：

```python
# 伪代码
import requests

headers = {
    "Authorization": f"WECHATPAY2-SHA256-RSA2048 {signature}",
    "Accept": "application/json",
}
response = requests.get(
    "https://api.mch.weixin.qq.com/v3/bill/tradebill?bill_date=2026-04-22",
    headers=headers,
    cert=("apiclient_cert.pem", "apiclient_key.pem"),
)
# 返回的是账单下载链接或重定向到 CSV 文件
```

### 2.3 结论

- **普通消费者**：无官方 API，只能手动导出 Excel 或通过邮箱接收账单链接
- **商户**：有完善的 API，但需要商户资质
- **推荐方案**：与支付宝/银行一样，通过微信 App 导出 CSV 后手动导入，或等待微信发送账单邮件到专用邮箱

## 3. 支付宝（Alipay）

### 2.1 官方开放 API

| 项目 | 详情 |
|------|------|
| 平台名称 | 支付宝开放平台（open.alipay.com） |
| API 文档 | https://opendocs.alipay.com/open/ |
| 核心接口 | `alipay.data.dataservice.bill.downloadurl.query` |
| 接口功能 | 查询对账单下载地址（日账单/月账单） |
| 数据格式 | CSV（ZIP 压缩） |
| 账单类型 | 交易账单、资金账单 |
| 权限要求 | 需创建应用并签约「账单查询」权限 |
| 个人可用 | 是，个人开发者可创建应用 |
| 认证方式 | RSA2 签名 + AppID |

### 2.2 账单 API 使用流程

```
1. 登录支付宝开放平台
2. 创建「网页&移动应用」
3. 生成 RSA2 密钥对（应用私钥 + 支付宝公钥）
4. 添加功能「账单查询"
5. 调用 bill.downloadurl.query 接口
6. 下载返回 URL 中的 ZIP 文件
7. 解压获取 CSV 账单
```

### 2.3 Python SDK

```bash
pip install alipay-sdk-python
```

### 2.4 账单 CSV 字段

| 字段 | 说明 |
|------|------|
| 支付宝交易号 | 唯一标识 |
| 商户订单号 | 商家系统的订单号 |
| 交易创建时间 | yyyy-MM-dd HH:mm:ss |
| 付款时间 | 实际扣款时间 |
| 最近修改时间 | 退款等操作时间 |
| 交易来源地 | 网站/APP |
| 类型 | 交易/退款/转账 |
| 交易对方 | 商家名称或个人账号 |
| 商品名称 | 交易描述 |
| 金额（元） | 正负表示收支 |
| 收/支 | 收入/支出 |
| 交易状态 | 成功/关闭/退款 |
| 服务费（元） | 手续费 |
| 成功退款（元） | 退款金额 |
| 备注 | 用户备注 |
| 资金状态 | 已收入/已支出 |

### 2.5 限制与注意事项

- 日账单：T+1 生成（昨天数据今天可查）
- 月账单：T+3 生成
- 账单保留时间：通常为 1 年
- 免费额度：足够个人使用
- 需要实名认证的支付宝账号

| 发送频率 | 每日/每周/每月 |
| 邮件格式 | HTML 表格或附件（部分卡种支持） |
| 数据内容 | 交易时间、金额、商户、余额 |
| 自动化 | 可通过邮件协议（IMAP/POP3）自动拉取 |

| 导出格式 | Excel / CSV |

| 自动化 | 需手动操作，无法定时自动导出 |

#### 方案 C：一网通网页版

| 项目 | 详情 |
|------|------|
| URL | https://cmbchina.com |
| 功能 | 可查询和导出交易明细 |

## 5. 淘宝 / 天猫（Taobao/Tmall）

### 4.1 官方开放 API（TOP）

| 项目 | 详情 |
|------|------|
| 平台名称 | 淘宝开放平台（TOP） |
| 文档 | https://open.taobao.com/doc.htm |
| 核心接口 | `taobao.trades.sold.get` |
| 功能 | 查询卖家已卖出的交易列表 |
| 权限要求 | App Key + App Secret + 用户授权 |
| 个人可用 | 可申请，但只能获取自己授权的订单 |

### 4.2 关键接口列表

| 接口名 | 功能 |
|--------|------|
| `taobao.trades.sold.get` | 获取已卖出的交易列表 |
| `taobao.trade.fullinfo.get` | 获取单笔交易的详细信息 |
| `taobao.trades.sold.increment.get` | 增量获取交易（适合定时同步） |

### 4.3 限制

- **只能获取自己作为买家的订单**：需要买家视角的接口
- `taobao.trades.bought.get` 是买家视角接口（可用性需确认）
- OAuth2 授权，access_token 有效期通常为 24 小时
- 应用需要审核，个人应用可能被限制
- 部分字段可能脱敏（如收货地址）

### 4.4 替代方案：网页抓取

使用 Playwright/Selenium 登录淘宝网页版，抓取「已买到的宝贝」页面数据。

**风险**：
- 淘宝有完善的反爬机制（滑块验证码、IP 限制）
- 违反用户协议
- 需要处理登录态维持（Cookie/Session）

## 8. 通用替代方案对比

### 7.1 邮件对账单（最可行的自动化方案）

| 渠道 | 支持邮件对账单 | 格式 | 自动化难度 |
|------|---------------|------|-----------|
| 支付宝 | 部分支持（月度账单邮件） | HTML | 低 |
| 招商银行 | 支持 | HTML/文本 | 低 |
| 工商银行 | 支持 | HTML | 低 |
| 建设银行 | 支持 | HTML | 低 |
| 中国银行 | 支持 | HTML | 低 |
| 京东/淘宝/PDD | 不支持 | - | - |

邮件对账单自动化流程：

```
1. 在银行/App 中开启邮件对账单
2. 使用 Python imaplib 连接邮箱
3. 按发件人/主题筛选对账单邮件
4. 解析 HTML/附件内容
5. 提取交易记录写入数据库
```

### 7.2 支付渠道间接获取

对于电商平台（京东/淘宝/PDD），交易最终通过支付宝或微信支付完成。可以通过支付渠道的账单间接获取消费记录：

| 支付渠道 | 交易描述信息 | 商户名称 |
|---------|-------------|---------|
| 支付宝 | 较完整（含商家名） | 清晰 |
| 微信支付 | 较完整 | 清晰 |
| 银行卡直付 | 可能简化 | 可能显示为「快捷支付」 |

### 7.3 手动导出 + 导入

| 渠道 | 导出格式 | 频率 |
|------|---------|------|
| 支付宝 | CSV/Excel | 随时 |
| 招商银行 App | Excel | 随时 |
| 京东 | 无直接导出 | - |
| 淘宝 | 无直接导出 | - |
| 拼多多 | 无直接导出 | - |

## 9. 推荐实现策略

### 8.1 短期方案（MVP 可用）

**优先级 1：支付宝账单 API**
- 最成熟的个人可用 API
- CSV 格式标准化
- 已调研字段可直接映射到系统模型

**优先级 2：银行邮件对账单**
- 覆盖招行等主流银行
- 通过 IMAP 自动拉取
- 需编写 HTML 解析器

**优先级 3：手动 CSV 导入**

- 各平台网页/App 导出后上传

## 10. 风险评估

| 风险 | 等级 | 说明 |
|------|------|------|
| API 权限变更 | 中 | 平台可能调整开放策略 |
